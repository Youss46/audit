## Z-Score and ratios (scoring-engine.ts)
Z-Score uses the classic Altman coefficients (1.2/1.4/3.3/0.6/0.999) mapped onto SYSCOHADA DSF line codes (BK, BD, BJ, DJ, DV, CH, DD, XI, XC, XD, TA, TB, CD, CB) pulled from the existing `computeDsf` result rather than re-deriving ledger figures. Thresholds: Z>2.9 → FAIBLE_RISQUE, 1.23–2.9 → RISQUE_MODERE, ≤1.23 → RISQUE_ELEVE.

Judgment calls (no explicit spec, documented as code comments at the point of use): solvency ratio = Capitaux Propres / Total Bilan; retained earnings pulled from the DSF line since no separate ledger account isolates it.

## Valuation assumptions
- Patrimoniale (ANR) = raw Capitaux Propres — no asset-revaluation data exists in this system, so no markup/markdown is applied.
- EBITDA-multiple = EBE (DSF line XC, used as the EBITDA proxy) × accountant-chosen multiplier (slider, default 6x).
- A third "capitalized earnings" figure (netIncome / capitalizationRate, default 10%) is shown as a bonus transparency view alongside the two headline methods, not a spec requirement.
- Default valuation scenario auto-inserts on first GET via `onConflictDoNothing` + re-read (race-safe upsert-on-read pattern).

## pdfmake 0.3 broke the whole app's PDF export API — fixed in export-engine.ts
`pdfmake` was upgraded (or pinned) to 0.3.x at some point, which silently changed the Node.js server API: the package's main export is no longer a `PdfPrinter` class you `new` up — it's now `module.exports = new pdfmake()`, a pre-configured singleton. The old pattern (`new PdfPrinterCtor(fonts)` then `.createPdfKitDocument(docDef)` returning an event-emitting stream) throws `"PdfPrinterCtor is not a constructor"`.

**New API:** call `printer.setFonts(fonts)` once, then `printer.createPdf(docDef).getBuffer()` (returns a Promise<Buffer> directly, no manual stream/chunks handling needed). Must also call `setLocalAccessPolicy(() => true)` — local file access must be allowed because pdfmake resolves the standard 14 fonts (Helvetica, etc.) through the same local-access check; leaving it denied breaks rendering with `"Access to local file denied"` even for built-in fonts. `setUrlAccessPolicy(() => false)` is safe to deny since no document definition in this codebase ever references a remote image URL.

**Why this matters:** this is shared infrastructure (`getPrinter()`/`renderPdf()` in `export-engine.ts`) used by every PDF export in the app (balance, financial statements, report documents, scoring executive summary) — the break and the fix both apply globally, not just to the module being built when it was discovered. If a PDF export ever throws a constructor or file-access error again, check `pdfmake`'s installed version/API shape before assuming the bug is in the calling code.

## Orval naming gotcha
An OpenAPI request-body schema named `SetValuationInput` in `openapi.yaml` gets code-generated by Orval as the Zod const `SetValuationBody` (not `SetValuationInput` — that name only exists as a TS type alias, which is not importable as a runtime schema). Always check the generated file for the actual exported const name rather than assuming it matches the OpenAPI schema name.
